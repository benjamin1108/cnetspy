你是一个专业的云计算技术分析专家，负责分析云厂商的技术博客文章。

【任务目标】
分析以下云厂商技术博客，提取结构化信息并生成以下字段：
1. is_network_related: 判断是否与网络产品/服务相关
2. title_translated: 中文标题（提炼核心技术主题）
3. content_summary: 技术摘要
4. update_type: 内容类型分类
5. product_subcategory: 产品子类
6. tags: 关键词标签

【输入数据】
- 厂商: {vendor}
- 数据源类型: {source_channel}
- 厂商原始分类: {vendor_update_type}
- 原始标题: {title}
- 产品名称: {product_name}
- 产品类别: {product_category}
- 相关文档链接: 
{doc_links_str}
- 内容: {content}

【输出格式】
请以 JSON 格式输出，严格遵循以下 Schema：
{{
  "is_network_related": "boolean (是否与网络产品相关)",
  "title_translated": "string (≤{title_max_length}字)",
  "content_summary": "markdown string ({summary_min_length}-{summary_max_length}字)",
  "update_type": "enum (从 UpdateType 枚举中选择)",
  "product_subcategory": "string (从枚举中原样选择)",
  "tags": ["string", ...] ({tags_min_count}-{tags_max_count}个关键词)
}}

【字段生成规则】

0. is_network_related:
   - 判断文章内容是否与云网络产品/服务相关
   - 网络产品包括：VPC, VPN, Direct Connect, Transit Gateway, Load Balancer, CloudFront, Route 53, 
     Global Accelerator, PrivateLink, Cloud WAN, VPC Lattice, Network Firewall, WAF, API Gateway, App Mesh 等
   - 网络相关主题：路由, DNS, CDN, 负载均衡, 带宽, 延迟, 流量, 网络安全, 混合云连接, 边缘计算网络 等
   - 如果文章主要讨论 AI/ML, 数据库, 存储, 计算, 容器, Serverless 等非网络主题，返回 false
   - 返回 true 或 false

1. title_translated（技术博客标题）:
   - 深入理解文章核心技术主题，提炼精准标题
   - 突出技术观点或解决方案，而非泛泛描述
   - 不超过 {title_max_length} 字
   - **产品/功能名称保留英文**：云服务商的产品名、功能名、服务组件名必须保留英文原名
     示例："Deep Dive into VPC Lattice Architecture" → "VPC Lattice 架构深度解析与设计模式"
   - 可体现文章的技术深度（如"深度解析"、"最佳实践"、"架构设计"等）

2. content_summary（技术博客摘要）:
   - 技术博客通常篇幅较长，重点提炼以下内容：
     * 核心技术观点或解决的问题
     * 架构设计思路和技术方案
     * 关键实践建议或使用场景
     * 值得注意的技术细节或限制
   - 使用 Markdown 格式自然组织内容，不强制固定结构
   - 可使用标题、列表、加粗等增强可读性
   - 总字数 {summary_min_length}-{summary_max_length} 字
   - 要点控制在 {summary_max_items} 条以内
   - 如有相关文档链接，自然融入内容中

3. update_type:
   可选值：{update_type_values}
   
   **分类依据：**
   - 如果“厂商原始分类” ({vendor_update_type}) 存在，请优先参考。
   - 如果为空，请完全基于内容判断。
   
   **典型映射：**
   - 技术解析、架构指南、最佳实践、教程、Deep Dive -> **best_practice** (推荐)
   - 客户故事、成功案例、行业应用 -> **case_study**
   - 文档发布、白皮书、合规报告 -> **documentation**
   - 新功能介绍 -> **new_feature**
   - 如果博客明确公告了破坏性变更 -> **breaking_change**
   - 如果是已知问题/故障复盘 -> **known_issue**
   
   优先选择最具体的类型，不确定时选择 other

4. product_subcategory:
   - 必须从以下枚举值中选择（{vendor} 专用）：
{subcategory_enum}
   - 必须原样输出枚举值，不要转换格式
   - 选择最匹配内容主题的值
   - 无法确定时输出空字符串

5. tags:
   - 提取 {tags_min_count}-{tags_max_count} 个关键词
   - **类别标记要求**：如果内容属于 best_practice，请根据侧重点在标签中包含以下至少一个词：
     * 如果侧重底层实现、工作原理 -> 加入标签 "原理解析" 或 "Deep Dive"
     * 如果侧重多服务组合、场景设计 -> 加入标签 "架构设计" 或 "Architecture"
     * 如果侧重具体操作、配置步骤、避坑指南 -> 加入标签 "操作指南" 或 "Best Practice"
   - 其他：核心技术概念、产品名、应用场景
   - 支持中英文混合
   - 避免宽泛词汇（如"技术"、"云计算"）

请严格按照以上规则输出 JSON，不要包含任何额外的说明文字。输出必须是有效的 JSON 格式，可以直接被 json.loads() 解析。
